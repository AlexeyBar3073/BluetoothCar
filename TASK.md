# Задача: Калибровка сигналов датчика скорости по GPS

## 1. Цель
Юстировка реального количества импульсов на 1 метр пути путем сравнения данных датчика и пройденного расстояния, рассчитанного по координатам GPS.

## 2. Пользовательский сценарий (UI/UX)
1. **Экран настроек**: В элементе "Сигналы датчика скорости" появляется кнопка или режим "Калибровка".
2. **Старт**: При нажатии кнопки "Начать калибровку":
    - Программа отправляет в БК команду: `calibrate_speed_start`.
    - Система фиксирует начальную точку GPS.
    - Статус элемента меняется на: "Режим калибровки: [X] м".
3. **Движение**: Пользователь начинает движение. Приложение в реальном времени накапливает дистанцию (в метрах) на основе перемещения GPS.
4. **Завершение**: При повторном нажатии на карточку открывается диалоговое окно:
    - Отображается пройденное расстояние по GPS.
    - Имеется поле для ручной корректировки расстояния (согласно одометру автомобиля).
    - Кнопка "Завершить калибровку".
5. **Финализация**: По кнопке "Завершить":
    - Отправляется JSON-команда: `{"command":"calibrate_speed_end","data":{"distance_m": <значение>}}`.
    - БК самостоятельно пересчитывает константу импульсов и сохраняет её.
    - Приложение выходит из режима калибровки.

## 3. Технические требования
- **Разрешения**: Проверка и запрос `ACCESS_FINE_LOCATION` (API 24+).
- **Точность**: Фильтрация "шумов" GPS (игнорирование точности > 15м, игнорирование дрейфа при нулевой скорости).
- **Состояние**: Хранение состояния калибровки в `SharedViewModel` для обеспечения непрерывности процесса при смене экранов.

## 4. Команды протокола
- **Начало**: `{"command":"calibrate_speed_start"}` (или строковый аналог по согласованию).
- **Конец**: `{"command":"calibrate_speed_end","data":{"distance_m":<int>}}`
