    * запрещено перефразировать

    * писать историю изменений

    * кода

    * менять поведение кода

   → НЕ выполнять изменения
   → запросить подтверждение у пользователя

# 📝 ОБЩИЕ ПРАВИЛА ОФОРМЛЕНИЯ

**Первая строка файла**
   Всегда должна содержать путь к файлу в ветке проекта
      Пример:
         НЕПРАВИЛЬНО: C:/Project/BluetoothCar/app/src/main/java/com/alexbar3073/bluetoothcar/data/bluetooth/listeners/ConnectionFeasibilityChecker.kt
         ПРАВИЛЬНО: //ФАЙЛ: data/bluetooth/listeners/ConnectionFeasibilityChecker.kt.kt



**Блок описания файла** [следует после импортов]

   Должен присутствовать блок `/** ... */`, содержащий:

    * ТЕГ [упрощённое название]
      * ФАЙЛ
      Пример:
         НЕПРАВИЛЬНО: C:/Project/BluetoothCar/app/src/main/java/com/alexbar3073/bluetoothcar/data/bluetooth/listeners/ConnectionFeasibilityChecker.kt
         ПРАВИЛЬНО: //ФАЙЛ: data/bluetooth/listeners/ConnectionFeasibilityChecker.kt.kt
    * МЕСТОНАХОЖДЕНИЕ
    * НАЗНАЧЕНИЕ ФАЙЛА
    * ОТВЕТСТВЕННОСТЬ
    * АРХИТЕКТУРНЫЙ ПРИНЦИП
    * КЛЮЧЕВОЕ ПРИНЦИП
    * СВЯЗИ С ДРУГИМИ ФАЙЛАМИ
