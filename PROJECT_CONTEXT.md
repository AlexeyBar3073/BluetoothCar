# Контекст проекта: BluetoothCar

## Текущее состояние
- **Стабилизация протокола**: ЗАВЕРШЕНА. 
- **DataStreamHandler**: Исправлен критический баг с `replay(1)`, приводивший к ложным подтверждениям команд после переподключения. Код оформлен по стандарту `AGENTS.md`.
- **OTA-обновление**: Реализовано по принципу **Lock-Step**. Отправка чанков прошивки (256 байт) происходит строго по одному с ожиданием подтверждения `ota_read` от устройства.
- **Безопасность OTA**: Внедрена проверка CRC16 для каждого чанка и таймауты на инициализацию, завершение и перезагрузку.

## Технические детали
- **Транспорт**: JSON через RFCOMM (Bluetooth SPP).
- **Механизм надежности**: 
  - Транспортный уровень: `msg_id` / `ack_id`.
  - Прикладной уровень (OTA): `ota_data` / `ota_read` + CRC16.
- **Минимальная версия**: Android 7.0 (API 24).

## Последние важные изменения (10.05.2026)
1. Откат `DataStreamHandler.kt` к стабильной архитектуре Flow и удаление кэширования в `acknowledgmentsFlow`.
2. Публичный доступ к `clearQueue()` для синхронизации состояния очереди при ошибках.
3. **Реализовано обнуление телеметрии**: При каждом переподключении и сбросе контроллера данные `CarData` сбрасываются в начальное состояние, чтобы избежать отображения неактуальной информации.
4. Добавлена детальная документация в коде на русском языке.
